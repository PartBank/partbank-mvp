## P03 — Database Schema + RLS + Seed

```
Read CONTEXT.md first, then proceed.

Create the complete Supabase database schema. Output this as a single SQL file at supabase/schema.sql

The SQL must include, in this exact order:

1. Enable UUID extension:
   CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

2. Create ENUM types:
   - user_role: 'customer', 'workshop', 'internal'
   - order_status: all 14 statuses from CONTEXT.md section 5 (use the exact string values)
   - manufacturability_grade: 'A', 'B', 'C', 'D'
   - part_status: 'request_only', 'ready_to_make'
   - workshop_tier: 'Bronze', 'Silver', 'Platinum'

3. Create tables (in dependency order):
   profiles, truck_brands, truck_models, part_categories, parts, workshops, orders, order_events, notifications, files

   profiles:
   - id UUID references auth.users PRIMARY KEY
   - role user_role NOT NULL
   - full_name TEXT
   - phone TEXT
   - created_at TIMESTAMPTZ DEFAULT now()

   workshops:
   - id UUID DEFAULT uuid_ossp() PRIMARY KEY
   - profile_id UUID references profiles NOT NULL
   - name TEXT NOT NULL
   - address TEXT
   - capability_tags TEXT[] DEFAULT '{}'
   - tier workshop_tier DEFAULT 'Bronze'
   - is_verified BOOLEAN DEFAULT false
   - created_at TIMESTAMPTZ DEFAULT now()

   truck_brands:
   - id, name, logo_url, created_at

   truck_models:
   - id, brand_id (FK truck_brands), name, year_range, created_at

   part_categories:
   - id, model_id (FK truck_models), name, created_at

   parts:
   - id, category_id (FK part_categories), name, description
   - manufacturability_grade, status part_status DEFAULT 'request_only'
   - material_spec TEXT, notes TEXT
   - drawing_file_path TEXT (Supabase Storage path)
   - created_by UUID references profiles
   - created_at TIMESTAMPTZ DEFAULT now()

   orders:
   - id UUID DEFAULT uuid_ossp() PRIMARY KEY
   - customer_id UUID references profiles NOT NULL
   - part_id UUID references parts NOT NULL
   - workshop_id UUID references workshops (nullable, set on assignment)
   - status order_status DEFAULT 'pending_re_confirmation'
   - quantity INTEGER DEFAULT 1
   - notes TEXT
   - re_fee INTEGER (in IDR)
   - part_price INTEGER (in IDR)
   - tracking_number TEXT
   - qc_failure_notes TEXT
   - created_at TIMESTAMPTZ DEFAULT now()
   - updated_at TIMESTAMPTZ DEFAULT now()

   order_events:
   - id, order_id (FK orders), actor_id (FK profiles)
   - from_status order_status, to_status order_status
   - notes TEXT, created_at TIMESTAMPTZ DEFAULT now()

   notifications:
   - id, user_id (FK profiles), order_id (FK orders nullable)
   - message TEXT NOT NULL
   - is_read BOOLEAN DEFAULT false
   - created_at TIMESTAMPTZ DEFAULT now()

   files:
   - id, order_id (FK orders), uploader_id (FK profiles)
   - file_type TEXT ('re_receipt', 'part_receipt', 'drawing', 'reference_photo')
   - storage_path TEXT NOT NULL
   - created_at TIMESTAMPTZ DEFAULT now()

4. Create updated_at trigger for orders table.

5. Row Level Security — enable on all tables, then add policies:

   profiles:
   - SELECT: auth.uid() = id OR role = 'internal'
   - INSERT: auth.uid() = id
   - UPDATE: auth.uid() = id

   orders:
   - SELECT customer: auth.uid() = customer_id
   - SELECT workshop: EXISTS (workshop assigned to this order where profile_id = auth.uid())
   - SELECT internal: role = 'internal'
   - INSERT: role = 'customer'
   - UPDATE: role = 'internal' (all updates go through internal or server actions)

   notifications:
   - SELECT: auth.uid() = user_id
   - INSERT: role = 'internal' OR server (use service role in API routes)

   parts:
   - SELECT: authenticated users
   - INSERT/UPDATE/DELETE: role = 'internal'

   workshops:
   - SELECT: authenticated users
   - INSERT: role = 'workshop'
   - UPDATE: role = 'internal'

   files:
   - SELECT: owner OR assigned workshop (status >= in_production) OR internal
   - INSERT: authenticated users

6. Seed data (INSERT statements):
   - 3 truck brands: Hino, Mitsubishi Fuso, Isuzu
   - 2 models each (6 total): e.g. Hino → FM 260, Ranger
   - 2 categories each model: e.g. Sistem Kemudi, Sistem Rem
   - 2–3 parts each category (all status: request_only)
   - 3 demo user profiles (use fixed UUIDs so they're consistent):
     Internal: full_name 'Admin PartBank', role 'internal'
     Workshop: full_name 'Bengkel Maju Jaya', role 'workshop'
     Customer: full_name 'Pak Budi Santoso', role 'customer'

Output ONLY the SQL file. No explanations.
```
